Design Document

Project Overview

This web application is designed to manage inclusive services pricing for treatments and services. It focuses on providing pricing information for radiology instruments that accommodate abnormal body types and comprehensive detoxification services.

Features and Functionalities

  • Home Page: Introduces the system with an updated splash screen image and provides navigation to all sections.
  • Add Pages: Separate forms in addService.html and addTreatment.html allow users to add new records with labels and required fields.
  • Manage Pages: Tables in manageService.html and manageTreatment.html display the first three records from the respective datasets.
  • Navigation: Consistent navigation menus on every page ensure easy access to the various sections.
  • Styling: The my_styles.css file is used to provide collapsed inside borders and alternating row colors for tables, without using the colgroup element.

User Interface Design

The interface is designed to be simple and user-friendly:

  • Data tables are styled for clarity and ease of reading.

Database Considerations

Although this submission focuses on the front-end, the application is intended to integrate with a database that holds two tables: one for Services and one for Treatments. Each table will have at least five fields and multiple records.

Conclusion

This design document outlines the structure and functionality of the Inclusive Services Pricing & Treatment Management system. Future enhancements may include search functionality, record editing, and deletion features.
