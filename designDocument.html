Design Document

Project Overview

This application manages inclusive services pricing, focusing on radiology instruments that accommodate abnormal body types and detoxification services.

Features and Functionalities

  • Home Page: Uses an updated splash screen image and introduces the app’s theme.
  • Add Pages: addService.html and addTreatment.html provide forms with required fields for each column in the “Services” and “Treatments” tables.
  • Manage Pages: manageService.html and manageTreatment.html show the first three records from each table with no colgroup elements.
  • Navigation: Each page has consistent navigation, so users can quickly switch between viewing and adding records.
  • Styling: my_styles.css ensures collapsed inside borders and table stylings, with alternating row backgrounds for clarity.

User Interface Design

The UI is simple, with a clear header, a navigation menu, and main content sections. Each form has labels and required attributes to ensure complete data input.

Database Considerations

Although this submission focuses on the front-end, the underlying database would contain two tables: “Services” and “Treatments,” each with at least five fields and five records.

Conclusion

This document details the structure and design of the Inclusive Services Pricing & Treatment Management system. Future enhancements might include search capabilities, record editing, and record deletion.
